# Copilot Memory Log

## 2026-05-21

- Investigated `Snapshot`/`Screenshot` flow and traced empty `Focused Window`/`Opened Windows` to `Desktop.get_state()` fast path for `use_ui_tree=False`, which intentionally set `windows=[]` and `active_window=None`.
- Verified this data is rendered by `_snapshot_helpers.capture_desktop_state()` via `desktop_state.windows_to_string()` and `desktop_state.active_window_to_string()`, so the empty lists are a direct consequence of the fast path.
- Implemented a surgical fix in `Desktop`:
  - Added `get_windows_fast()` using `win32gui.EnumWindows` + `GetWindowText` + `GetWindowRect` + `GetWindowThreadProcessId`.
  - Added `get_active_window_fast()` using `win32gui.GetForegroundWindow`.
  - Added lightweight helpers to build `Window` models from raw handles without UIA traversal.
  - Updated `get_state(use_ui_tree=False)` to populate windows/active window from these fast Win32 calls.
- Updated existing tests in `tests/test_snapshot_display_filter.py` to reflect the new fast-path behavior and ensure the non-UI-tree code path now returns active/open windows.
- Added a dedicated regression test (`test_get_state_keeps_open_windows_when_use_ui_tree_false`) to verify open windows remain populated when the UI tree path is disabled.
- Ran parallel validation; addressed a code review optimization note by removing unnecessary full window enumeration from `get_active_window_fast()` when no precomputed window list is supplied.
- Environment notes:
  - `uv` is not available in this sandbox.
  - Attempted targeted pytest run failed during collection on Linux because `sys.getwindowsversion` is unavailable; this test module requires Windows runtime.
  - Full desktop-service tests cannot be executed here due missing Windows-only modules (`win32gui`, `win32process`, `win32con`), but test updates were kept minimal and scoped to the changed behavior.
